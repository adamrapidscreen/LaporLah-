# E2-S3: Build Responsive Nav Wrapper

## Description
Create a responsive navigation wrapper component that switches between the mobile bottom navigation bar (<768px) and the desktop top navigation bar (≥768px). Integrate the wrapper into the root layout so navigation is present on all pages.

## Dependencies
- Stories: E2-S1, E2-S2
- Files: `src/components/layout/bottom-nav.tsx`, `src/components/layout/top-nav.tsx`, `src/app/layout.tsx`

## Tasks
1. `src/components/layout/nav-wrapper.tsx`: Create Client Component that renders `BottomNav` on mobile and `TopNav` on desktop using Tailwind responsive classes (`hidden md:block` / `md:hidden`)
2. `src/app/layout.tsx`: Integrate `NavWrapper` into the root layout, add bottom padding for mobile nav clearance (`pb-20 md:pb-0`), wrap main content area

## Implementation Notes

### Nav Wrapper (`src/components/layout/nav-wrapper.tsx`)
```typescript
'use client';

import { usePathname } from 'next/navigation';

import { BottomNav } from './bottom-nav';
import { TopNav } from './top-nav';

const HIDE_NAV_ROUTES = ['/login'];

export function NavWrapper() {
  const pathname = usePathname();

  // Hide navigation on certain routes (e.g., login page)
  if (HIDE_NAV_ROUTES.includes(pathname)) {
    return null;
  }

  return (
    <>
      {/* Desktop: Top Nav */}
      <div className="hidden md:block">
        <TopNav />
      </div>

      {/* Mobile: Bottom Nav */}
      <div className="md:hidden">
        <BottomNav />
      </div>
    </>
  );
}
```

### Updated Root Layout (`src/app/layout.tsx`)
```typescript
import { NavWrapper } from '@/components/layout/nav-wrapper';

// Inside the body:
<ThemeProvider attribute="class" defaultTheme="dark" enableSystem disableTransitionOnChange>
  <NavWrapper />
  <main className="min-h-screen pb-20 md:pb-0">
    {children}
  </main>
</ThemeProvider>
```

### Responsive Breakpoint Strategy
- **Mobile (<768px)**: Bottom nav fixed, main content has `pb-20` to clear 64px nav + 16px buffer
- **Desktop (≥768px)**: Top nav sticky, main content has `pb-0`
- Uses CSS-based responsive switching (no JS media queries) for SSR compatibility

### Route-Based Hiding
Navigation is hidden on specific routes like `/login` where a full-screen layout is desired. The `HIDE_NAV_ROUTES` array is easily extensible.

## Acceptance Criteria
- [ ] On viewports <768px, only the bottom navigation is visible
- [ ] On viewports ≥768px, only the top navigation is visible
- [ ] Navigation is hidden on the `/login` route
- [ ] Main content area has `pb-20` on mobile for bottom nav clearance
- [ ] Main content area has `pb-0` on desktop
- [ ] `NavWrapper` integrated into root layout
- [ ] No layout shift when switching between breakpoints
- [ ] CSS-based responsive switching (Tailwind `hidden md:block`) — no JS media queries
- [ ] Component uses `'use client'` directive
- [ ] Named export: `NavWrapper`
