# E10-S5: Mobile Responsiveness Audit

## Description
Audit and fix all pages for mobile responsiveness at 375px width. Ensure proper touch targets (minimum 44px), safe area insets for bottom navigation, consistent side padding (16px), and no overflow/truncation/misalignment issues across the entire application.

## Dependencies
- Stories: E1 through E9 (all features must be built)
- Files: All page and component files

## Tasks
1. **Home Feed** (`src/app/page.tsx`, `src/components/reports/report-card.tsx`):
   - Verify cards stack vertically with 16px side padding
   - Verify category filter is horizontally scrollable without overflow
   - Ensure touch targets on filter buttons ≥ 44px
2. **Report Detail** (`src/app/report/[id]/page.tsx`):
   - Verify photo fills width at 16:9 ratio
   - Verify map is full-width and doesn't overflow
   - Ensure all action buttons (follow, flag, share) have ≥ 44px touch targets
   - Verify comment section doesn't overflow
3. **Create Report** (`src/app/create/page.tsx`):
   - Verify form inputs are full-width on mobile
   - Verify map picker is usable on touch
   - Ensure submit button is easily tappable
4. **Profile** (`src/app/profile/page.tsx`):
   - Verify stats grid doesn't overflow
   - Verify badges wrap correctly
   - Ensure report list cards are full-width
5. **Notifications** (`src/app/notifications/page.tsx`):
   - Verify notification items have adequate touch targets
   - Ensure long text truncates with ellipsis, not overflow
6. **Admin Dashboard** (`src/app/admin/page.tsx`, `src/app/admin/flagged/page.tsx`, `src/app/admin/users/page.tsx`):
   - Verify stat cards grid is 2 columns on mobile
   - Verify user table is horizontally scrollable if needed
   - Ensure action buttons stack or wrap on narrow screens
7. **Login** (`src/app/login/page.tsx`):
   - Verify centered layout with adequate padding
   - Ensure OAuth buttons are full-width and tappable
8. **Global Layout** (`src/app/layout.tsx`):
   - Verify bottom nav has `pb-[env(safe-area-inset-bottom)]`
   - Verify no horizontal scroll on any page
   - Ensure consistent `px-4` (16px) side padding on all containers

## Implementation Notes

### Common Mobile Fixes
```tsx
// Safe area for bottom navigation
<nav className="fixed bottom-0 left-0 right-0 border-t bg-background pb-[env(safe-area-inset-bottom)]">

// Prevent horizontal overflow
<main className="overflow-x-hidden">

// Minimum touch target
<button className="min-h-[44px] min-w-[44px]">

// Consistent side padding
<div className="px-4">

// Truncate long text
<p className="truncate">

// Responsive grid
<div className="grid grid-cols-2 md:grid-cols-3 lg:grid-cols-6">
```

### Viewport Meta (should exist in layout.tsx)
```tsx
export const metadata: Metadata = {
  viewport: 'width=device-width, initial-scale=1, viewport-fit=cover',
};
```

### Known Mobile Patterns to Apply
- Tables → Use card-based layout or horizontal scroll wrapper on mobile
- Grid → `grid-cols-1 sm:grid-cols-2` for card grids
- Dialog → `max-w-[calc(100vw-32px)]` on mobile
- Map → `w-full` with fixed height
