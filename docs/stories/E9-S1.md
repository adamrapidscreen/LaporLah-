# E9-S1: Build Admin Layout with Role Guard

## Description
Create the admin layout with Supabase role-based access control that redirects non-admin users to the home page. Include tab navigation for Flagged, Reports, and Users sections.

## Dependencies
- Stories: E1-S3 (auth/Supabase client), E1-S2 (user role in users table)
- Files: `src/lib/supabase/server.ts`, `src/app/layout.tsx`

## Tasks
1. `src/app/admin/layout.tsx`: Create Server Component layout that:
   - Gets current user session via `createServerClient()`
   - Queries `users` table for `role` column where `id` matches session user
   - Redirects to `/` via `redirect()` if not authenticated or role !== 'admin'
   - Renders page title "Admin Dashboard" and tab navigation (Flagged, Reports, Users)
   - Wraps `{children}` in a padded container
2. `src/app/admin/page.tsx`: Create placeholder Server Component that exports default page (will be replaced by E9-S2)

## Implementation Notes

### Layout with Role Guard
```tsx
// src/app/admin/layout.tsx
import { redirect } from 'next/navigation';
import { createServerClient } from '@/lib/supabase/server';
import Link from 'next/link';

export default async function AdminLayout({ children }: { children: React.ReactNode }) {
  const supabase = await createServerClient();
  const { data: { user } } = await supabase.auth.getUser();

  if (!user) redirect('/');

  const { data: profile } = await supabase
    .from('users')
    .select('role')
    .eq('id', user.id)
    .single();

  if (profile?.role !== 'admin') redirect('/');

  return (
    <div className="min-h-screen bg-background">
      <div className="border-b">
        <div className="container mx-auto px-4 py-4">
          <h1 className="text-2xl font-bold">Admin Dashboard</h1>
        </div>
        <nav className="container mx-auto px-4">
          <div className="flex gap-4 border-b">
            <Link href="/admin/flagged" className="px-3 py-2 text-sm font-medium hover:text-primary border-b-2 border-transparent hover:border-primary transition-colors">
              Flagged
            </Link>
            <Link href="/admin" className="px-3 py-2 text-sm font-medium hover:text-primary border-b-2 border-transparent hover:border-primary transition-colors">
              Reports
            </Link>
            <Link href="/admin/users" className="px-3 py-2 text-sm font-medium hover:text-primary border-b-2 border-transparent hover:border-primary transition-colors">
              Users
            </Link>
          </div>
        </nav>
      </div>
      <main className="container mx-auto px-4 py-6">
        {children}
      </main>
    </div>
  );
}
```

### Navigation Pattern
- Use Next.js `Link` component for client-side navigation between admin tabs
- Active tab styling can be enhanced with `usePathname()` in a Client Component nav wrapper if desired
- Tab order: Flagged (priority view), Reports (overview), Users (management)
